Get started

The CHU (Centre Hospitalier Universitaire) API of Caen provides programmatic access to manage their database (Materiel, Employee Tables). This API allows to use CRUD Method on each table (Create, Read, Update, Delete).

To use this API, you need to be a CHU Employee.

Contact us at charley.lebarbier@isen-ouest.yncrea.fr to get your own access.

MATERIEL

The 'Materiel' table get be accessed with the following url :
http://localhost/api/materiel


TABLE COMPOSITION

Champ Type Description
id int (Auto increment, non modifiable, non accessible), identifie le matériel avec un id unique
hardware_name String Nom du matériel
hardware_size String Dimension du matériel
hardware_condition String Etat du matériel

EMPLOYEE

The 'Employee' table get be accessed with the following url :
http://localhost/api/employee


TABLE COMPOSITION

Champ Type Description
id int (auto increment, non modifiable, non accessible), Identifie l'employé.e avec un id unique
last_name String Nom de famille de l'employé.e
first_name String Prénom de l'employé.e
age Int Age de l'employé.e
job_position String Poste de l'employé.e

FUNCTIONS

The CHU API uses the following functions (available for both table):

Function Description
get all • /api/materiel to get all element in 'materiel' table
• /api/employee to get all element in 'employee' table
get by id example : /api/(materiel or employee)/23
get the id element specified from 'materiel or employee' table
add use json format to add :
• 'materiel' parameters : hardware_name, hardware_size, hardware_condition
• 'employee' parameters : last_name, first_name, age, job_position
update use json format to update :
• 'materiel' parameters : id, hardware_name, hardware_size, hardware_condition
• 'employee' parameters : id, last_name, first_name, age, job_position
delete use json format to delete material / employee with parameter : id
